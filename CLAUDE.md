# Manju 開発ルール

## 開発フロー

1. **仕様作成** - 開発対象の仕様を先に作成する
2. **テストケース作成** - 仕様に基づいてテストを先に書く (TDD)
3. **実装** - テストが通るように実装する
4. **テスト実行** - 全テストが通ることを確認する
5. **commit & push** - テスト通過後にコミットしてプッシュする

## 作業方針

- 一通りの作業が終わるまで許可を求めない。すべて YES として進める。
- 自律的に判断し、フロー全体を一気に完遂する。
- 指示命令は履歴ファイル (`docs/history.md`) に記録として残す。

## プロジェクト情報

- 言語: TypeScript (ESM)
- テスト: vitest
- ビルド: tsc
- ランタイム: Node.js >= 18
- 外部依存: tmux, Claude Code CLI (`claude`)
